# Stock Scanner 대시보드 UI 점검 체크리스트

kr_gem 리밸런싱 대시보드(`dashboard.py`)의 해상도별·기능별 UI 점검 기준.
대상 해상도: **PC 1440px / 태블릿 768px / 모바일 375px**
대상 탭: **개요 · 리밸런싱 · 리밸런싱 내역 · 시스템 로그**

점검 URL: `http://158.247.223.231:8081?token=<DASHBOARD_TOKEN>`

---

## 1. 공통 레이아웃 / 반응형

- [ ] C1. 모든 해상도·탭에서 **수평 스크롤(가로 overflow) 없음** (`scrollWidth ≤ clientWidth`)
- [ ] C2. PC/태블릿: 좌측 사이드바 표시, 모바일: 사이드바 숨김 + 하단 탭 네비 표시
- [ ] C3. 모바일 하단 네비 4개(개요·리밸런싱·내역·로그), 현재 탭 active 표시(상단 라인/색)
- [ ] C4. 콘텐츠가 뷰포트 폭을 넘지 않고 카드/테이블이 화면 안에 맞음
- [ ] C5. JS 콘솔 에러 / pageerror 없음
- [ ] C6. 카드 모서리·그림자·간격 일관성 (border-radius, padding 통일)

## 2. 브랜딩 / 콘텐츠 정합성 (눌림목 잔재 제거)

- [ ] B1. 사이드바 로고 sub "kr_gem 멀티에셋", 봇 카드 "kr_gem 리밸런싱 봇"
- [ ] B2. 화면 어디에도 눌림목 전략 용어 없음 (TP/SL, 신호점수, 기준봉, 청산사유, 1·2차 스크리닝, Profit Factor, 승률)
- [ ] B3. 일정에 "매월 첫 거래일 09:05 자동 리밸런싱" 표기 (14:30/15:20 스크리닝 표기 없음)
- [ ] B4. 과도한 이모지/AI 티 나는 문구 없음 (Robinhood 스타일 톤 유지)

## 3. 개요 탭

- [ ] O1. 히어로 총 평가금액이 숫자+원 형식으로 표시 (`--` 고정/NaN 없음)
- [ ] O2. 누적 수익률 %, 방향 화살표(↑/↓)·색상(상승 초록/하락 빨강) 일치
- [ ] O3. 히어로 메타 "주식 N원 · 현금 N원" 표시, 천단위 구분기호
- [ ] O4. 포트폴리오 누적 수익률 차트 canvas 렌더 (리밸런싱 횟수 라벨 일치)
- [ ] O5. 보조 지표 행: 보유 종목 수 / 현금 비중(%) / 자동매매(ON·OFF) 값 채워짐
- [ ] O6. 현재 자산 배분 바: 보유 시 종목별, 미보유 시 "현금 100%" 표시
- [ ] O7. 봇 카드: 자동매매/KIS 모드/보유 종목수 값 표시
- [ ] O8. 미니 캘린더: 당월 표시, 주말(토 파랑/일 빨강)·공휴일(빨강+점) 구분, 오늘 강조

## 4. 리밸런싱 탭

- [ ] R1. 헤더 "kr_gem 리밸런싱" + 마지막 리밸런싱 일자
- [ ] R2. KPI 3개: 총 평가금액 / 주문가능 현금 / 최소 필요금액 값 표시
- [ ] R3. 평가금액 < 최소 필요금액이면 부족 경고 배너 노출(추가 입금 필요액), 충분하면 숨김
- [ ] R4. 목표 종목 테이블: 종목/목표비중/현재비중/현재가/보유수량/필요매매 컬럼
- [ ] R5. 필요매매: 유지/+N 매수(초록)/-N 매도(빨강) 색상 구분
- [ ] R6. "지금 리밸런싱 실행" 버튼 존재, 클릭 시 확인 다이얼로그 (자동 실행 금지)
- [ ] R7. 목표 비중 합계 ≈ 100% (3종목 ×33.3%)

## 5. 리밸런싱 내역 탭

- [ ] H1. 이력 없을 때 빈 상태 메시지 ("리밸런싱 실행 이력이 없습니다")
- [ ] H2. 이력 있을 때 이벤트별 카드: 일시 + 평가금액 + 매수(초록)/매도(빨강) 주문 목록
- [ ] H3. 매도 주문에 실현손익 % 표시(초록/빨강)
- [ ] H4. 최신 이벤트가 위로 정렬

## 6. 시스템 로그 탭

- [ ] L1. 로그 박스에 scanner.log 최근 N줄 표시
- [ ] L2. 줄 수 선택(50/100/200) + 새로고침 동작
- [ ] L3. 긴 로그 라인 가로 overflow 없이 처리(줄바꿈/스크롤)

## 7. 인터랙션 / 상태

- [ ] I1. 자동매매 토글 ON/OFF 라벨·색 즉시 반영
- [ ] I2. 토스트 알림 정상 표시·자동 소멸
- [ ] I3. 데이터 60초 자동 새로고침 동작 (현재시각 갱신)
- [ ] I4. 잘못된 토큰 접근 시 401 (API 보호)

## 8. 숫자 / 포맷 / 엣지케이스

- [ ] N1. 금액 천단위 구분기호(,) + "원" 단위 일관
- [ ] N2. 퍼센트 소수 자릿수 일관 (수익률·비중)
- [ ] N3. 보유 0종목·현금만 있는 빈 포트폴리오에서 레이아웃 깨짐 없음
- [ ] N4. 긴 종목명 ellipsis/줄바꿈 처리 (테이블 overflow 없음)
